### [2026-02-25 20:00] fix(office): adjust office layout, pixel mirroring, and sleep animations
- **Changes:** Added `mirrorFrame()` function in `pixelData/parser.ts` for pixel-level horizontal mirroring. `PixelCharacter.tsx` now accepts a `mirrored` prop that mirrors frame data at the pixel level instead of using CSS `scaleX(-1)`. `AgentWorkstation.tsx` removed CSS `scaleX(-1)` flip in favor of pixel mirroring, adjusted `marginBottom` for tighter character-desk gap, and simplified sleeping `translateY`. `OfficeView.tsx` adjusted all agent y-coordinates downward (back row 155-160, mid 250-260, front 345-350). `OfficeRoom.tsx` changed wall/floor ratio from 45/55% to 30/70%, repositioned rug, coffee zone, and wainscoting. `pixelData/room.ts` moved wall decorations down to center on the shorter wall (y: 12-18 range). `ZzzParticles.tsx` repositioned from `-top-2 -right-1` to `top-0 right-0`. `app.css` removed rotate from sleep-nod animation and reduced zzz float distance to 12px.
- **Reason:** CSS `scaleX(-1)` caused sub-pixel rendering artifacts and inconsistent flip behavior across browsers; pixel-level mirroring produces crisp results. Layout ratios and coordinates were off, making agents float above desks and the room feel top-heavy. Tightening sleep animation values keeps visual effects subtle and grounded.
- **Files:** apps/desktop/src/components/office/pixelData/parser.ts, apps/desktop/src/components/office/PixelCharacter.tsx, apps/desktop/src/components/office/AgentWorkstation.tsx, apps/desktop/src/components/office/OfficeView.tsx, apps/desktop/src/components/office/OfficeRoom.tsx, apps/desktop/src/components/office/pixelData/room.ts, apps/desktop/src/components/office/ZzzParticles.tsx, apps/desktop/src/app.css

### [2026-02-25 18:00] feat(chat): add markdown rendering and conversation context
- **Changes:** Agent chat messages now render full markdown (headers, code blocks, tables, lists, blockquotes, links, horizontal rules) via `react-markdown` + `remark-gfm`. User messages remain plain text. XSS protection added through a custom `components` prop that blocks `javascript:` and `data:` URI schemes, allowing only `http`/`https` links. - **Reason:** Raw text rendering made agent responses hard to read when they contained structured content (code, tables, lists). Conversation context ensures agents can reference prior exchanges for coherent multi-turn interactions.
