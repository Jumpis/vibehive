### [2026-02-25 18:00] feat(chat): add markdown rendering and conversation context
- **Changes:** Agent chat messages now render full markdown (headers, code blocks, tables, lists, blockquotes, links, horizontal rules) via `react-markdown` + `remark-gfm`. User messages remain plain text. XSS protection added through a custom `components` prop that blocks `javascript:` and `data:` URI schemes, allowing only `http`/`https` links. Comprehensive `.chat-markdown` CSS styles added covering typography, code formatting, tables, blockquotes, and links. Chat store enhanced with `buildTaskWithContext()` that prepends up to 20 recent completed messages as conversation history to API calls, enabling context continuity across turns.
- **Reason:** Raw text rendering made agent responses hard to read when they contained structured content (code, tables, lists). Conversation context ensures agents can reference prior exchanges for coherent multi-turn interactions.
- **Files:** apps/desktop/src/components/chat/ChatMessageBubble.tsx, apps/desktop/src/stores/chatStore.ts, apps/desktop/src/app.css, apps/desktop/package.json, apps/desktop/pnpm-lock.yaml

### [2026-02-25 17:15] feat(chat): add agent direct chat panel with sliding drawer UI
- **Changes:** New sliding drawer panel for direct agent communication. Added `ChatMessage` type, `chatStore` (Zustand) with `openChat`/`closeChat`/`sendMessage`/`addAgentResponse` actions, and 5 chat UI components: `AgentChatPanel` (drawer container with backdrop), `ChatHeader` (agent identity + status), `ChatMessageList` (with auto-scroll and empty state), `ChatMessageBubble` (user/agent message styling with status indicators), and `ChatInput` (textarea with Cmd/Ctrl+Enter shortcut). Modified `App.tsx` to mount the panel at root level, `AgentCard.tsx` and `AgentWorkstation.tsx` to open chat on click (replacing tooltip in office view), `useWebSocket.ts` to route `mission_completed` events for direct missions into the chat store. Added slide-in CSS animation. Applied bug fixes: `crypto.randomUUID()` for message IDs, `aria-label` on close button, `navigator.userAgent` instead of deprecated `navigator.platform`, `EMPTY_MESSAGES` constant to prevent infinite re-renders.
- **Reason:** Enable users to interact directly with individual agents through a chat-like interface, providing a more intuitive and conversational UX for issuing direct tasks and viewing results, rather than relying on tooltips or the mission form.
- **Files:** apps/desktop/src/types/chat.ts (new), apps/desktop/src/stores/chatStore.ts (new), apps/desktop/src/components/chat/AgentChatPanel.tsx (new), apps/desktop/src/components/chat/ChatHeader.tsx (new), apps/desktop/src/components/chat/ChatInput.tsx (new), apps/desktop/src/components/chat/ChatMessageBubble.tsx (new), apps/desktop/src/components/chat/ChatMessageList.tsx (new), apps/desktop/src/App.tsx, apps/desktop/src/app.css, apps/desktop/src/components/dashboard/AgentCard.tsx, apps/desktop/src/components/office/AgentWorkstation.tsx, apps/desktop/src/hooks/useWebSocket.ts

### [2026-02-25 16:47] feat(office): add agent sleep animation for idle workstations
- **Changes:** Agents now visually fall asleep after 30 seconds of idle status. Sleep behavior includes a head-nod CSS animation with brightness dimming, floating "zzz" particle overlay, and halved sprite frame rate. Clicking a sleeping agent or a status change wakes them instantly. Added `useSleepTimer` hook to manage idle timeout, `ZzzParticles` component for the floating text effect, `SLEEP_IDLE_THRESHOLD_MS` constant, and new CSS keyframes (`pixel-sleep-nod`, `pixel-zzz-float`).
- **Reason:** Give users an immediate visual cue that an agent has been idle for a while, reinforcing the "living office" metaphor and making active vs inactive agents distinguishable at a glance.
- **Files:** apps/desktop/src/hooks/useSleepTimer.ts (new), apps/desktop/src/components/office/ZzzParticles.tsx (new), apps/desktop/src/components/office/PixelCharacter.tsx, apps/desktop/src/components/office/AgentWorkstation.tsx, apps/desktop/src/lib/constants.ts, apps/desktop/src/app.css
